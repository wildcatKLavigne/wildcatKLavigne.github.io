Project Management for Engineering Teams

Students learn how engineering teams organize work, communicate progress, and anticipate failure before it happens. Unit 2 establishes the team charter, project plan, and risk register that govern every subsequent unit. All three Project Management tools are then actively used — not just studied — throughout the rest of the course.

Class Lessons
Class 6
Roles, Responsibilities & Team Norms
Introduces the four engineering team roles (Build Lead, Documentation Lead, Parts Manager, Quality Lead) and the team charter. Teams negotiate role assignments, write and sign their charter, then immediately apply all Unit 1 skills to a real engineering challenge.
🔩 Activity: REV Starter Bot rotary intake — design, build & test a flapper module (all Unit 1 skills applied)
Open worksheet Notebook template Worksheet .docx

Learning objectives

  • Name and describe the four team roles and explain what each role owns
  • Write a signed team charter with specific, enforceable norms
  • Apply EDP, sketching, weighted matrix, and modular design to the flapper challenge using team roles
Novice: Fill in charter template; analyze 2 pre-drawn flapper designs; complete matrix with provided criteria
Intermediate: Write charter from scratch; sketch 3 different flappers; complete weighted matrix with own criteria
Proficient: Design a mid-build deviation protocol for the charter; connect flapper to modular design properties from Unit 1
Class 7
Project Planning: Milestones, Timelines & Task Breakdown
Introduces Work Breakdown Structure (WBS), Gantt charts, task dependencies, and the critical path concept. Teams decompose the entire remaining course into a concrete project plan — phases, tasks, owners, and time estimates — and build a Gantt timeline that they will update in every subsequent unit.
📅 Activity: Build a full-course WBS and Gantt chart for Classes 9–40
Open worksheet Notebook template Worksheet .docx

Learning objectives

  • Decompose a project into tasks, subtasks, owners, and time estimates using a WBS
  • Build a Gantt chart showing task duration, dependencies, and the critical path
  • Define milestones with specific, verifiable completion criteria and owner assignments
Novice: Fill in WBS subtasks and owners; shade 10+ Gantt cells; answer Q1 with teacher support
Intermediate: Complete full WBS with definitions of done; complete Gantt with dependency arrows
Proficient: Identify critical path; calculate buffer-adjusted completion date; PERT analysis on most uncertain task
Class 8
Risk Management, Progress Tracking & Team Communication
Introduces the risk register: identify, score (likelihood × impact), and mitigate project risks before they happen. Teams practice writing a specific status update (traffic light + facts + action items) and run a structured stand-up meeting. Closes Unit 2 with a Gantt update and first written status report.
⚠️ Activity: Build an 8-risk register + write the Unit 2 status update
Open worksheet Notebook template Worksheet .docx

Learning objectives

  • Build a risk register: identify 8+ risks, score each L×I, assign mitigations and owners
  • Write a specific status update: traffic light + facts + owner + action items
  • Run a structured stand-up in under 3 minutes; park blockers without derailing the meeting
Novice: Identify 5 risks using the provided example row; write one mitigation per risk
Intermediate: 8 risks across 4 categories; detailed mitigation for all Critical (score 7–9) risks
Proficient: Preventive + contingency mitigation for top 2 risks; FMEA extension on one robot subsystem
Team Roles Reference

The four team roles are introduced in Class 6 and rotate each unit so every student practices every skill by the end of the course. Keep this reference in your notebook.

🔧 Build Lead
Owns physical construction: build sequence, sub-task assignment, real-time build decisions, deviation recording, squareness verification, build sign-off.
📓 Documentation Lead
Owns the engineering notebook: current entries, sketch standards (dimensions, labels, title blocks, dates), revision tracking, stand-up notes, teacher review preparation.
📦 Parts & Inventory Manager
Owns parts: inventory tracking, material sourcing from stockpile, parts list maintenance, shortage flagging before build blockers occur, workspace management.
🔬 Quality & Test Lead
Owns test plans: success criteria written before building, structured test execution, quantitative result recording, root-cause identification, iteration decision driving.
Project Management Tools Summary

Three tools from Unit 2 are used throughout the course. Refer to this summary whenever you update these documents in later units.

Tool What it answers Updated when Owner role
Team charter How do we make decisions and resolve conflict? Reviewed at role rotation; amended by consensus Documentation Lead
WBS + Gantt chart What needs to happen, when, and who owns it? Start of Units 5, 6, 8; whenever milestone targets change Documentation Lead (records); all roles (input)
Risk register What might go wrong, and how do we prevent it? Before every major build phase; when a risk materializes Quality Lead (owns); all roles identify risks
Stand-up meeting What did I do, what am I doing, what is blocking me? Every class, first 3 minutes Documentation Lead (facilitates + records)
Status update Are we on track? What is the current blocker? End of every unit Documentation Lead (writes); all roles provide facts
