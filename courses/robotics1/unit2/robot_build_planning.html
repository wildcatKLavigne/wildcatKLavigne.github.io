Learning Objectives

  1. Decompose the full robot build into phases, tasks, subtasks, owners, and time estimates using a Work Breakdown Structure — using your real experience from earlier deliverables to calibrate estimates.
  2. Build a course-level Gantt chart that spans Units 3–9, shows task dependencies, identifies the critical path, and assigns role owners to every task group.
  3. Define milestones for each phase: specific, verifiable conditions that tell the team a phase is truly complete — not "mostly done."
  4. Identify at least one task on the critical path and explain what the team should do if that task slips by one class period.
Why this session comes after Class 9: You have now practised WBS and Gantt tools on two real deliverables — the flapper design (Class 6) and the chassis build (Classes 7–8) — and you have completed your first hardware inventory and structural design work in Class 9. You know what it actually takes to define a problem, research materials, sketch designs, and build something. That experience makes your estimates here grounded in reality, not guesswork. A robot build plan written before you've built anything is a wish list. A plan written now is a project plan.
Part 1  ·  Calibrate Your Estimates Before You Plan (10 min)

What Your Earlier Work Tells You About Your Estimates

The most dangerous project plan is one where every task is estimated by people who have never done anything like it before. The second most dangerous is one where the team ignores what they learned from practice. You have data from two real builds. Before writing a single cell in the WBS, use that data to set realistic expectations for the tasks ahead.

The three anchors that should shape every estimate in this session: (1) how long Define and Research took on the chassis build relative to how much time you planned for them; (2) how long Prototype took relative to plan — this is almost always the most underestimated stage; and (3) how long Iterate took — which you could not predict in advance at all. Carry those patterns into every phase of the plan you build today.

Part 3  ·  Course Gantt Chart (20 min)

Mapping Every Task to a Timeline

The Gantt chart turns your WBS into a schedule. Each row is a task group from the WBS. Each column pair represents a unit. You shade the cells where that task is active. Where one task must finish before another starts, leave a gap — that gap is a dependency arrow. The critical path is the longest unbroken chain of shaded, dependent cells from left to right. Every task on the critical path gets a bold border.

The documentation row spans the entire chart because the Documentation Lead has continuous responsibility throughout the build — but it is not on the critical path because it doesn't gate any build task. Showing it as parallel reinforces the lesson from Class 6: the Documentation Lead role is always active, never in a sprint at the end.

Part 4  ·  Milestone Definitions (12 min)

How the Team Knows a Phase Is Truly Complete

A milestone is a gate. You do not start Phase 4 before Phase 3's milestone is confirmed. "Chassis built" is not a milestone — it is a description of activity. "Chassis passes squareness test (diagonals within 2mm); Quality Lead signed off; all motors respond to driver input; drive test completed and build log updated — Class 20" is a milestone. Notice: it has a measurable test, a named owner, and a target class number. If the team arrives at Class 21 and the milestone isn't met, they know exactly what is blocking them and who owns the fix.

Extension / Exploration

Research PERT (Program Evaluation Review Technique) — the probabilistic scheduling tool used on the Apollo program and the Polaris missile project. Apply a full PERT analysis to the two tasks in your WBS with the highest variance ratios from Part 1: for each, estimate O (optimistic), M (most likely), and P (pessimistic) class periods, and calculate E = (O + 4M + P) ÷ 6. Then calculate the PERT standard deviation: σ = (P − O) ÷ 6. Write 3 sentences explaining: (1) what a high σ tells you about a task, (2) how you would use σ to decide how much buffer to allocate, and (3) which of your six phases has the highest total uncertainty and why.
