Engineering Foundations

Students build the mental models and documentation habits that underpin all future engineering work: intentional design, the Engineering Design Process, sketching and multi-view drawing, and modular thinking. These concepts are reinforced throughout the course.

4 classes (Classes 1–4)
74 min per class
Groups of 4
Next: Unit 2 — Project Management
Key deliverable
Engineering Notebook, Classes 1–4
Design rule introduced
3 sketches before you build
Frameworks
EDP, Decision Matrix, Modular Architecture
Scaffolding levels
Novice · Intermediate · Proficient
Class Worksheets
Class 1
What is Engineering? Intentional Design
Discussion, robot photo analysis, and engineering notebook setup. Students examine visible design decisions in real robot photos and make their first notebook entry.
Open worksheet Download .docx

Learning objectives

  • Define intentional design and distinguish it from trial-and-error
  • Identify design decisions in a robot photo with supporting rationale
  • Set up and make the first entry in an engineering notebook
Novice: Guided worksheet — analyze 3 design choices from a provided photo
Intermediate: Compare two robots; write a comparison paragraph
Proficient: Write a design critique; propose one improvement with rationale
Class 2
The Engineering Design Process
Introduce the 6-stage EDP loop. Students map a familiar product through all stages and practice generating multiple design alternatives using the "3 sketches" rule.
Open worksheet Download .docx

Learning objectives

  • Name and describe all 6 EDP stages
  • Apply the EDP to a familiar product by mapping each stage to real decisions
  • Explain why iteration is a planned feature of engineering, not a failure
Novice: Fill in an EDP graphic organizer for a provided scenario
Intermediate: Create own EDP diagram with branching iteration paths
Proficient: Reflect on FTC build season — what stages do teams skip and why?
Class 3
Engineering Documentation, Sketching & Multi-View Drawing
Orthographic views, dimension labeling, revision boxes, and the "3 sketches" rule applied to REV motor mount designs. Students select their preferred design using a decision matrix.
Open worksheet Download .docx

Learning objectives

  • Sketch an object in 2+ orthographic views with dimension labels
  • Generate 3 motor mount alternatives and evaluate with a decision matrix
  • Format notebook entries correctly: date, signature, page number, revision box
Novice: Trace bracket outline; add dimension labels using a template
Intermediate: Freehand front and top views; complete decision matrix
Proficient: Full 3-view sketch + isometric; weighted decision matrix + written justification
Class 4
Modular Design Thinking
REV's own modular system as a case study. Students reverse-engineer the modularity of a competition robot and design a 5-module architecture for their own robot — a blueprint used throughout the semester.
Open worksheet Download .docx

Learning objectives

  • Define modular design and identify interfaces between modules
  • Explain the benefits and costs of modularity
  • Propose a 5-module robot architecture defining inputs, outputs, and constraints
Novice: Label 4 modules on a robot diagram; describe what each does
Intermediate: Identify interfaces; sketch one interface detail; draw the architecture diagram
Proficient: Define inputs/outputs/constraints per module; identify parallel-buildable modules
